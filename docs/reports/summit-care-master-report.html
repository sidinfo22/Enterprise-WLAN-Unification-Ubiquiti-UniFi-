Executive Focus

What This Project Is Designed To Prove

Summit Care Medical Clinic now operates many offices throughout the five boroughs. The project proves that Ubiquiti UniFi can give the clinic one repeatable platform for branch routing, secure staff Wi-Fi, guest access, cloud monitoring, remote provisioning, VPN, and wireless coverage.

Unified Management

The Cloud Key Gen2 hosts UniFi Network so IT can adopt devices, see topology, monitor clients, push SSIDs, update firmware, and manage sites through local private access or unifi.ui.com.

Enterprise Staff Wireless

Staff Wi-Fi authenticates through Windows Server NPS/RADIUS using UDP 1812 for authentication and UDP 1813 for accounting, tying access to Active Directory users and groups. Managed Summit Care devices can also use certificate-based access so trusted endpoints are validated before joining the staff WLAN.

Guest Isolation

Guest Wi-Fi uses a separate VLAN, subnet, firewall policy, and UniFi captive portal so patients and visitors receive internet access without reaching internal clinic resources.

Important design note: this is a WLAN unification and cloud management project, not just an access point installation. The value comes from tying routing, wireless, RADIUS, guest policy, VPN, monitoring, and provisioning into one repeatable UniFi model.

Architecture & Configuration

Cloud Key, Gateway, RADIUS, VLANs & Wireless Mesh

The technical build centers on three layers: the UniFi gateway for routing, firewalling, VLANs and VPN; the Cloud Key Gen2 for network management; and Windows Server NPS/RADIUS for staff identity authentication.

Core Network Design

Network Example VLAN/Subnet Purpose
Management VLAN 10 / 10.10.10.0/24 Cloud Key, gateway, switches, AP management, and administrator interfaces
Staff Wi-Fi VLAN 20 / 10.20.20.0/24 Domain-authenticated employee devices
Clinical devices VLAN 30 / 10.30.30.0/24 Approved clinic systems and office equipment
Guest Wi-Fi VLAN 40 / 10.40.40.0/24 Internet-only access through captive portal controls

Cloud Key Gen2 Setup

  • Connect the Cloud Key Gen2 to the management network and reserve a private management IP.
  • Open the local GUI for first-time setup and controller hardening.
  • Link authorized cloud access through unifi.ui.com and require protected admin accounts.
  • Update firmware, create site naming standards, configure backups, and adopt UniFi devices.

Windows NPS/RADIUS Setup

  • Install Network Policy Server on the Windows Server VM running on Proxmox.
  • Register NPS in Active Directory and create policies for approved Summit Care staff groups.
  • Add UniFi infrastructure as RADIUS clients with shared secrets.
  • Permit UDP 1812 and UDP 1813 between UniFi and Windows Server.
  • Enable certificate-based 802.1X access, such as EAP-TLS or PEAP with certificate validation, for managed Summit Care devices.

Certificate-Based UniFi Access

Certificate-based access strengthens the staff SSID by validating the device, not just the person typing a password. Summit Care can issue certificates from the Windows certificate authority to approved domain-joined devices. During the UniFi 802.1X login flow, the U7 Pro XL access points forward the request to Windows NPS, NPS validates the certificate and policy, and UniFi places the device onto the proper staff VLAN only after authentication succeeds.

Guest Portal and Wireless Mesh

The guest SSID lands on a dedicated VLAN and subnet, uses a UniFi captive portal with Summit Care branding, and is blocked from private internal networks. For one two-floor office without Ethernet to the second floor, wireless meshing was enabled so a second-floor U7 Pro XL could uplink to a wired access point and extend coverage where cabling was not available.

Operations, VPN & Monitoring

Cloud Operations, VPN, Monitoring & Provisioning

Summit Care can manage UniFi locally through private management addresses or remotely through unifi.ui.com. VPN access gives authorized administrators a secure path back into approved internal networks without exposing management interfaces directly to the internet.

Private GUI Access

Administrators can open the Cloud Key or gateway by private IP while on site or connected through VPN, which supports local recovery and direct management.

Cloud Access

UniFi cloud access provides remote visibility into offices, devices, clients, alerts, topology, firmware, and adoption state from one place.

VPN Access

The UniFi gateway provides VPN connectivity using the appropriate method, such as Teleport, WireGuard, or L2TP/IPsec based on support needs.

Operational Runbook

  • Open UniFi Network through unifi.ui.com or the private controller address.
  • Review gateway health, WAN status, AP status, switch uplinks, VLAN assignment, and client experience.
  • For staff Wi-Fi issues, verify RADIUS reachability to Windows Server NPS on UDP 1812 and UDP 1813, then confirm certificate trust for managed devices.
  • For guest issues, test captive portal load, guest VLAN addressing, DNS, firewall restrictions, and internet egress.
  • For mesh issues, review wireless uplink quality, retries, channel utilization, and second-floor AP placement.
Risk, Governance & Conclusion

Security Controls, Governance & Final Summary

Risk Impact Mitigation
RADIUS or domain controller outage Staff may be unable to authenticate to enterprise Wi-Fi Monitor NPS, protect the Windows Server VM, and document fallback access
Guest VLAN leakage Visitors could reach internal clinic assets Use explicit deny rules from guest to private networks and test segmentation
Cloud admin account compromise Remote UniFi settings could be changed maliciously Require MFA, least privilege, admin review, and sensitive-change alerting
Mesh uplink weakness Second-floor performance may drop if backhaul quality is poor Monitor uplink quality and add Ethernet cabling when feasible
Final summary: the project demonstrates a complete UniFi WLAN unification design for Summit Care Medical Clinic. The UniFi gateway provides routing, security, and VPN; the Cloud Key Gen2 provides the management plane; U7 Pro XL access points deliver the wireless layer; Windows Server NPS connects staff Wi-Fi to domain identity and certificate-based access; and VLANs, guest portal controls, cloud monitoring, and mesh uplinks make the design practical across real clinic offices in the five boroughs.
